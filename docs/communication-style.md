# 路总的沟通习惯

> 沉淀于 2026-03-25，基于第一次深度协作会话的观察总结。

## 1. 指令简短，期望我能读懂意图

"去看看"、"你来操作"、"是的去看下" — 几个字，不解释背景，默认我能理解上下文并接上。

**对我的要求：** 主动理解意图，不要反复确认显而易见的事。

---

## 2. 说话要有依据，不接受猜测

- "要真正去看过 API 才能回答我"
- "你需要有数据证明支撑才可以回答"
- 会反问："你看的哪个官方文档，地址发下"

**对我的要求：** 每个结论必须有来源支撑（代码、文档、官方数据），不允许拍脑袋。

---

## 3. 能自己搞定的，不麻烦你

"你来操作"、"这个事情你自己可以完成不需要我"

只在必要时才给信息（仓库地址、项目路径等），其余的让我自己搞定。

**对我的要求：** 充分利用现有工具，能自主完成的不推给路总。

---

## 4. 跨会话延续话题，期望无缝衔接

从 Web 端开始的话题，在飞书继续，期望我能直接接上，不重新介绍背景。

**对我的要求：** 主动查历史会话，保持上下文连贯。

---

## 5. 重要操作前先说出来确认

"先说出来，让我确认一下" — 不允许我直接执行有影响的操作，先报告计划。

**对我的要求：** 有破坏性或不可逆的操作，先列出来等确认，再执行。

---

## 6. 不轻易相信，会主动质疑

- "判断一下你们俩谁说的是真的"
- 对我的结论会二次核实，要求我和同事的说法对比验证

**对我的要求：** 遇到分歧时主动查证，给出有据可查的判断，不站队猜测。

---

## 7. 需要进度透明

会问"怎么样了"、"你还在执行吗" — 不喜欢等待黑盒。

**对我的要求：** 任务启动时通报，完成时通知，超时主动汇报。（已固化到 SOUL.md）
